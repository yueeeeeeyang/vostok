Vostok

面向 JDK 17+ 的轻量 Java 框架，通过统一门面 Vostok 聚合十三个模块能力，一次初始化即可使用数据访问、Web 服务、缓存、日志、配置、安全、事件、游戏引擎、HTTP 客户端、AI 等全部功能。

快速开始

Maven 依赖

<!-- pom.xml -->
<dependency>
    <groupId>yueyang</groupId>
    <artifactId>vostok</artifactId>
    <version>1.9.0.4</version>
</dependency>

统一初始化

使用 Vostok.init(...) 在一个入口里按需初始化模块 — 配置了哪个模块就启动哪个，未配置的模块不会被触发。

import yueyang.vostok.Vostok;
import yueyang.vostok.data.VKDataConfig;
import yueyang.vostok.web.VKWebConfig;
import yueyang.vostok.log.VKLogConfig;
import yueyang.vostok.cache.VKCacheConfig;

Vostok.init(cfg -> cfg
    // 日志
    .logConfig(new VKLogConfig().outputDir("logs").consoleEnabled(true))

    // 数据库
    .dataConfig(new VKDataConfig()
        .url("jdbc:mysql://127.0.0.1:3306/demo")
        .username("root").password("123456")
        .maxActive(20))
    .dataPackages("com.example.entity")

    // 缓存
    .cacheConfig(new VKCacheConfig()
        .providerType("MEMORY").maxEntries(10000))

    // Web 服务器
    .webConfig(new VKWebConfig().port(8080))
    .webSetup(web -> web
        .get("/hello", (req, res) -> res.ok("Hello, Vostok!"))
    )
    .webStart(true)
);
提示
除统一入口外，各模块仍可独立使用 Vostok.Data.init(...)、Vostok.Web.init(...) 等方式初始化。

初始化顺序

模块按以下顺序启动；关闭时按相反顺序执行。

Config → Log → Security → Data → Cache → Event → Game → Http → File → AI → Web

关闭

// 关闭所有模块（逆序）
Vostok.close();

注意事项

  • Data：调用前必须先 init(...)，实体类须标注 @VKEntity。
  • Web：调用 start() 前必须先 init(...) 并注册路由。
  • Cache：不依赖 Config 模块，必须通过 VKCacheConfig 显式初始化。
  • Log / Config：支持懒加载，首次使用时会自动以默认配置初始化。
  • Security：主动调用型，不会自动接入 Data/Web 执行链路。
  • Game：关闭自动 ticker 后需业务侧手动调用 tickOnce()。
  • Http：使用相对路径时，须先注册带 baseUrl 的命名 Client。
注意
当前版本定位为实验与技术验证，不建议直接用于生产环境。运行环境为 JDK 17+。

mvn package -DskipTests
