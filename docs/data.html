🗄️ Vostok.Data

Data 数据访问模块

基于 JDBC 的数据访问层，提供实体 CRUD、事务管理、查询构建器、聚合操作、多数据源切换和连接池。

初始化

Vostok.Data.init(new VKDataConfig()
    .url("jdbc:mysql://127.0.0.1:3306/mydb")
    .username("root")
    .password("123456")
    .driver("com.mysql.cj.jdbc.Driver")
    .maxActive(20)
    .minIdle(5),
    "com.example.entity"   // 实体扫描包，可传多个
);
说明
支持 MySQL、PostgreSQL、Oracle、DB2、SQL Server，框架通过 dialect 自动适配分页等方言差异。

实体定义

@VKEntity(table = "users")
public class User {
    private Long id;

}

CRUD 操作

新增

User user = new User();
user.setName("Alice");
user.setEmail("alice@example.com");
Vostok.Data.insert(user);

// 批量新增
List<User> users = List.of(user1, user2, user3);
Vostok.Data.batchInsert(users);

查询

// 按主键
User user = Vostok.Data.findById(User.class, 1L);

// 查全部
List<User> all = Vostok.Data.findAll(User.class);

// 条件查询
VKQuery q = VKQuery.where("email", "alice@example.com")
                    .limit(10).offset(0);
List<User> list = Vostok.Data.query(User.class, q);

更新 / 删除

user.setName("Bob");
Vostok.Data.update(user);

Vostok.Data.delete(User.class, 1L);

// 批量删除
Vostok.Data.batchDelete(User.class, List.of(1L, 2L, 3L));

查询构建器 VKQuery

VKQuery q = VKQuery
    .where("status", "active")
    .and("age", ">=", 18)
    .or("role", "admin")
    .orderBy("created_at", false)   // false = DESC
    .limit(20).offset(40)
    .groupBy("department")
    .having("count(*) > 5");

List<User> result = Vostok.Data.query(User.class, q);

指定返回列

List<User> partial = Vostok.Data.queryColumns(
    User.class, q, "id", "name"
);

聚合查询

VKQuery q = VKQuery.where("status", "active");

// COUNT
long total = Vostok.Data.count(User.class, q);

// 多维聚合
Map<String, Object> stats = Vostok.Data.aggregate(
    User.class, q,
    VKAggregate.count("id", "total"),
    VKAggregate.avg("age", "avg_age"),
    VKAggregate.max("score", "max_score")
);

事务

// Runnable — 无返回值
Vostok.Data.tx(() -> {
    Vostok.Data.insert(order);
    Vostok.Data.update(inventory);
});

// Supplier — 有返回值
Order saved = Vostok.Data.tx(() -> {
    Vostok.Data.insert(order);
    return Vostok.Data.findById(Order.class, order.getId());
});

多数据源

// 注册额外数据源
Vostok.Data.registerDataSource("analytics", new VKDataConfig()
    .url("jdbc:postgresql://analytics-db:5432/stats")
    .username("reader").password("secret"));

// 切换数据源执行
Vostok.Data.withDataSource("analytics", () -> {
    return Vostok.Data.findAll(Report.class);
});

异步线程传播

// 捕获当前上下文（数据源、事务等）
VostokContext ctx = Vostok.Data.captureContext();

CompletableFuture.runAsync(ctx.wrap(() -> {
    Vostok.Data.findAll(User.class);
}));

SQL 拦截器

Vostok.Data.registerInterceptor((sql, params, next) -> {
    long start = System.currentTimeMillis();
    Object result = next.execute();
    long cost = System.currentTimeMillis() - start;
    if (cost > 200) Vostok.Log.warn("慢 SQL: {} ms, {}", cost, sql);
    return result;
});

连接池监控

VKPoolMetrics m = Vostok.Data.poolMetrics();
System.out.printf("active=%d, idle=%d, waitQueue=%d%n",
    m.activeCount(), m.idleCount(), m.waitQueueSize());

配置参数

参数类型默认值说明
urlString—JDBC 连接 URL
usernameString—数据库用户名
passwordString—数据库密码
driverString自动推断JDBC 驱动类名
dialectVKDialect自动推断SQL 方言（MYSQL / POSTGRESQL / ORACLE / DB2 / SQLSERVER）
minIdleint2连接池最小空闲连接数
maxActiveint10连接池最大活跃连接数
maxWaitMslong3000获取连接最大等待时间（ms）
idleTimeoutMslong600000空闲连接超时时间（ms）
leakDetectMslong0连接泄漏检测阈值（0=关闭）
testOnBorrowbooleantrue借出连接时校验是否存活
slowSqlMslong1000慢 SQL 阈值（ms）
batchSizeint500批量操作每批大小
autoCreateTablebooleanfalse启动时自动建表
retryEnabledbooleanfalse操作失败后是否自动重试

API 速查

方法说明
insert(entity)插入单条记录
batchInsert(list)批量插入
update(entity)按主键更新
batchUpdate(list)批量更新
delete(Class, id)按主键删除
batchDelete(Class, ids)批量删除
findById(Class, id)按主键查询
findAll(Class)查询全部记录
query(Class, VKQuery)条件查询
queryColumns(Class, VKQuery, fields...)查询指定列
count(Class, VKQuery)统计数量
aggregate(Class, VKQuery, VKAggregate...)聚合查询
tx(Runnable)事务（无返回值）
tx(Supplier)事务（有返回值）
registerDataSource(name, config)注册命名数据源
withDataSource(name, action)切换数据源执行
captureContext()捕获上下文用于异步传播
registerInterceptor(interceptor)注册 SQL 拦截器
poolMetrics()获取连接池指标

} catch (VKTxException e) {
    // 事务错误
}
