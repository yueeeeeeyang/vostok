⚙️ Vostok.Config

Config 配置管理模块

统一配置访问，自动扫描 .properties / .yml / .yaml 文件，支持多优先级覆盖、文件热监听、${key} 插值、自定义验证器与解析器。

初始化

// 默认配置（自动扫描 classpath 下 *.properties/*.yml）
Vostok.Config.init();

// 自定义配置
Vostok.Config.init(new VKConfigOptions()
    .watchEnabled(true)           // 开启文件热监听
    .watchDebounceMs(300)         // 防抖延迟（ms）
    .loadEnv(true)               // 加载环境变量
    .loadSystemProperties(true)  // 加载 JVM 系统属性
    .extraScanDirs("/etc/myapp") // 额外扫描目录
);
懒加载
未显式 init() 时，首次读取配置项会以默认配置懒加载初始化。

读取配置

// 读取字符串
String host = Vostok.Config.getString("db.host");
String host = Vostok.Config.getString("db.host", "localhost");  // 含默认值

// 读取整数 / 布尔
int     port    = Vostok.Config.getInt("db.port", 3306);
boolean enabled = Vostok.Config.getBoolean("feature.enabled", false);

// 获取某前缀下所有 key-value
Map<String, String> dbMap = Vostok.Config.getMap("db");

// 获取所有 key
Set<String> keys = Vostok.Config.getAllKeys();

插值语法

配置文件中可用 ${key} 引用其他配置项，支持默认值语法 ${key:defaultValue}。

# application.properties
base.url=http://api.example.com
retry.url=${base.url}/retry
timeout=${REQUEST_TIMEOUT:5000}  # 从环境变量读，缺省 5000

配置优先级

从低到高，高优先级覆盖低优先级：

  1. classpath 配置文件
  2. extraScanDirs 目录下的文件
  3. userDir 目录下的文件
  4. 手动 addFile() 的文件
  5. 环境变量（loadEnv=true）
  6. JVM 系统属性（loadSystemProperties=true）
  7. 运行时 putOverride()

运行时覆盖

// 写入运行时覆盖（最高优先级）
Vostok.Config.putOverride("feature.darkMode", "true");
Vostok.Config.removeOverride("feature.darkMode");
Vostok.Config.clearOverrides();

手动追加配置文件

Vostok.Config.addFile("/etc/myapp/secrets.properties");
Vostok.Config.addFiles("/conf/a.yml", "/conf/b.yml");
Vostok.Config.clearManualFiles();  // 移除手动追加的文件

// 手动触发全量重载
Vostok.Config.reload();

文件变更监听

Vostok.Config.registerListener((changedKeys) -> {
    if (changedKeys.contains("server.port")) {
        System.out.println("port changed to: " + Vostok.Config.getInt("server.port", 8080));
    }
});

配置校验

Vostok.Config.registerValidator(config -> {
    String url = config.getString("db.url");
    if (url == null || url.isBlank()) {
        throw new IllegalStateException("db.url is required");
    }
});

自定义解析器

// 注册 .toml 文件解析器
Vostok.Config.registerParser(new VKConfigParser() {
    public boolean supports(String filename) { return filename.endsWith(".toml"); }
    public Map<String, String> parse(InputStream is) { /* … */ }
});

配置参数

参数类型默认值说明
watchEnabledbooleanfalse是否开启文件热监听
watchDebounceMslong300文件变更防抖延迟（ms）
loadEnvbooleanfalse是否加载环境变量
loadSystemPropertiesbooleanfalse是否加载 JVM 系统属性
extraScanDirsString[]—额外扫描目录路径
userDirString—用户配置目录（优先于 classpath）

API 速查

方法说明
getString(key)读取字符串（未找到返回 null）
getString(key, default)读取字符串（含默认值）
getInt(key, default)读取整数（含默认值）
getBoolean(key, default)读取布尔（含默认值）
getMap(prefix)读取前缀下所有配置为 Map
getAllKeys()获取所有已加载的 key 集合
addFile(path)手动追加配置文件
clearManualFiles()移除手动追加的文件
putOverride(key, value)写入运行时覆盖
removeOverride(key)移除运行时覆盖
clearOverrides()清空所有运行时覆盖
registerListener(listener)注册配置变更监听器
registerValidator(validator)注册配置校验器
registerParser(parser)注册自定义文件格式解析器
reload()强制重新加载所有配置
