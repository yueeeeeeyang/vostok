🎮 Vostok.Game

Game 游戏引擎模块

实时多人游戏服务端引擎，基于房间 Tick 驱动模型，提供分片线程池、热点房间迁移、帧同步广播、玩家生命周期、断线重连、匹配队列、统一消息中心等核心能力。

初始化

默认初始化

// 以默认配置初始化，返回 VostokGame 实例（用于调用实例方法）
VostokGame game = Vostok.Game.init();

自定义配置

VostokGame game = Vostok.Game.init(new VKGameConfig()
    .tickRate(20)                    // 每秒 20 Tick（50ms/tick）
    .tickWorkerThreads(8)            // Tick 工作线程数（默认 CPU 核数）
    .maxRooms(10000)
    .maxPlayersPerRoom(128)
    .antiCheatEnabled(true)
    .matchmakingEnabled(false)
);
实例方法与静态方法

init() 是静态方法，返回 VostokGame 单例实例。房间管理、玩家管理、指令提交、匹配、消息等均为实例方法，需通过 init() 返回的实例（或链式调用）来调用。监控、tickOnce、close 等运维方法为静态方法，可直接通过 Vostok.Game.xxx() 调用。

通过 Vostok.init() 一体初始化

Vostok.init(cfg -> cfg
    .gameConfig(new VKGameConfig().tickRate(30))
    .gameSetup(game -> {
        game.registerLogic("chess", myLogic);
        game.createRoom("lobby", "chess");
    })
);

注册游戏逻辑（VKGameLogic）

VKGameLogic 是核心接口，所有方法均有默认空实现，按需覆盖即可。注意：逻辑注册是实例方法，须通过 init() 返回的实例调用，支持链式调用。

VostokGame game = Vostok.Game.init(new VKGameConfig().autoStartTicker(false));

game.registerLogic("chess", new VKGameLogic() {

    /** 房间创建后第一次 Tick 前触发 */
    public void onRoomStart(VKGameRoom room) {
        Vostok.Log.info("room {} started", room.getRoomId());
    }

    /** 房间进入 DRAINING 状态时触发（优雅关闭流程开始） */
    public void onRoomDraining(VKGameRoom room, String reason) {
        Vostok.Log.info("room {} draining: {}", room.getRoomId(), reason);
    }

    /** 房间彻底关闭时触发 */
    public void onRoomClose(VKGameRoom room) {
        Vostok.Log.info("room {} closed", room.getRoomId());
    }

    /** 玩家加入时触发 */
    public void onPlayerJoin(VKGameRoom room, VKGamePlayerSession player) {
        Vostok.Log.info("player {} joined room {}", player.getPlayerId(), room.getRoomId());
    }

    /** 玩家正常离开时触发 */
    public void onPlayerLeave(VKGameRoom room, VKGamePlayerSession player) {}

    /** 玩家断线时触发（Session 保留，可在宽限期内重连） */
    public void onPlayerDisconnect(VKGameRoom room, VKGamePlayerSession player) {}

    /** 玩家重连成功时触发 */
    public void onPlayerReconnect(VKGameRoom room, VKGamePlayerSession player) {}

    /** 玩家指令通过反作弊校验、即将被处理时触发（在 onTick 之前逐条触发） */
    public void onCommand(VKGameRoom room, VKGameCommand command) {
        room.attributes().put("last_cmd", command.getType());
    }

    /** 每 Tick 调用一次，tickNo 从 1 开始单调递增（房间级计数） */
    public void onTick(VKGameRoom room, long tickNo) {
        // tickNo == room.getCurrentTick()（本帧执行后的值）
    }
});
回调方法触发时机
onRoomStart(room)房间创建后，第一次 Tick 前
onRoomDraining(room, reason)房间进入 DRAINING 状态时（drainRoom 或超时）
onRoomClose(room)房间彻底关闭时（DRAINING → CLOSED 或直接 removeRoom）
onPlayerJoin(room, player)玩家成功加入房间
onPlayerLeave(room, player)玩家正常离开（leave 调用）
onPlayerDisconnect(room, player)玩家断线（disconnect 调用），Session 保留
onPlayerReconnect(room, player)玩家断线重连成功
onCommand(room, command)指令通过反作弊校验后，在 Tick 线程中逐条触发
onTick(room, tickNo)每帧（Tick）调用一次，tickNo 从 1 开始

提交指令（VKGameCommand）

指令通过构造函数创建，无 Builder。提交后经反作弊校验，通过则入队，失败抛 VKGameException(COMMAND_REJECTED)；队列满时抛 QUEUE_FULL；房间处于 DRAINING 状态时抛 STATE_ERROR。

// 最简构造（playerId, type, payload），优先级默认 NORMAL
game.submit("room-42", new VKGameCommand("player-1", "MOVE", Map.of("x", 1, "y", 3)));

// 指定优先级
game.submit("room-42", new VKGameCommand("player-1", "ATTACK", payload, VKGameCommandPriority.HIGH));

// 带客户端序号（clientSeq），用于反作弊单调性校验
game.submit("room-42", new VKGameCommand("player-1", "MOVE", payload, /*clientSeq*/ 42L));

// 完整构造（playerId, type, payload, clientSeq, timestampMs, priority）
game.submit("room-42", new VKGameCommand(
    "player-1", "MOVE", payload,
    /*clientSeq*/ 42L, System.currentTimeMillis(),
    VKGameCommandPriority.NORMAL
));

// 便捷提交（roomId, playerId, type, payload），优先级默认 NORMAL
game.submit("room-42", "player-1", "MOVE", Map.of("x", 1));

// 非阻塞提交（队列满时返回 false，不抛异常）
boolean accepted = game.trySubmit("room-42", new VKGameCommand("player-1", "MOVE", payload));

VKGameCommand 字段说明

字段（getter）类型说明
getPlayerId()String发出指令的玩家 ID
getType()String指令类型（业务自定义字符串）
getPayload()Object指令载荷（业务自定义）
getClientSeq()long客户端序号（-1 表示未设置，不做单调性校验）
getTimestampMs()long客户端时间戳（epoch ms），反作弊时钟偏移校验
getPriority()VKGameCommandPriority队列优先级：HIGH / NORMAL / LOW

VKGameCommandPriority

枚举值说明
HIGH高优先级（输入/战斗关键帧），每轮优先消费
NORMAL普通优先级（默认）
LOW低优先级（可延迟的非关键事件）
指令调度顺序
引擎按权重轮转消费三个优先级队列：默认权重 HIGH=4 / NORMAL=2 / LOW=1（可通过 highPriorityWeight 等配置调整）。低优先级指令不会永久饥饿，当高优先级队列为空时低优先级指令优先补位。

反作弊（AntiCheat）

启用 antiCheatEnabled=true 后，每条指令提交时执行以下校验，失败则抛 VKGameException(COMMAND_REJECTED)：

校验项配置参数说明
频率限制maxCommandsPerSecondPerPlayer单玩家每秒最大指令数，超出拒绝
指令序号单调性enforceMonotonicClientSeqclientSeq 必须递增（设置了 clientSeq 时），防止重放攻击
时间戳偏移maxClientTimestampSkewMs客户端时间戳与服务端偏差超阈值则拒绝
在线玩家限制requireOnlinePlayerCommand为 true 时断线玩家的指令被拒绝

帧同步（Frame Sync）

帧同步模式下，每帧 onTick 完成后，引擎将本帧所有已处理的玩家输入打包为 VKGameFrame，通过 VKGameFrameNotifier 推送给应用层，由应用层负责广播给房间内所有客户端。

与 onTick / onCommand 不互斥
帧同步模式下 onTick 和 onCommand 回调仍然执行，可用于服务端权威状态维护和反作弊。帧广播与消息中心（MessageCenter）完全隔离，保证最低延迟，无重试、无持久化。 
// 1. 创建帧同步房间
VKGameRoom room = game.createRoom("room-fps", "fps", true);

// 2. 绑定帧广播回调（房间级）
// 回调在 Tick Worker 线程调用，应快速返回（建议仅做序列化 + 异步发送）
game.bindFrameNotifier("room-fps", frame -> {
    byte[] bytes = FrameSerializer.toBytes(frame);
    for (String playerId : getOnlinePlayers(frame.roomId())) {
        websocket.sendAsync(playerId, bytes);
    }
});

// 3. 断线重连客户端追帧（拉取历史帧）
// 返回从 fromFrameNo 开始最多 limit 帧；环形缓冲区只保留最近 frameSyncHistoryCapacity 帧
List<VKGameFrame> history = game.pollFrames("room-fps", fromFrameNo, 50);

// 4. 解绑回调
game.unbindFrameNotifier("room-fps");

VKGameFrame 字段

字段类型说明

serverTimestamp()long服务端处理该帧时的时间戳（epoch ms），用于网络延迟估算
inputs()List<VKGameFrameInput>本帧所有已处理的玩家输入；静默帧（无输入）时为空列表

VKGameFrameInput 字段

字段类型说明

消息中心

统一消息中心提供发布/订阅、推送通知、拉取、ACK 确认、过期清理、自动重推等能力，适用于聊天、系统通知、告警、惩罚等业务场景。

发布消息

// 使用静态工厂：玩家聊天（发布到房间维度，房间内所有玩家可见）
List<VKGameMessage> msgs = game.publishMessages(List.of(
    VKGameMessagePublishCommand.playerChat("room-42", "player-1", "hello!")
));

// 使用静态工厂：系统公告（ROOM 维度广播）
game.publishMessages(List.of(
    VKGameMessagePublishCommand.systemNotice(
        VKGameMessageScope.ROOM, "room-42", "赛季结束", "本局游戏即将结束")
));

// 完整构造（type, scope, scopeId, senderId, title, content, payload, expireAtMs）
// expireAtMs=0 表示使用默认 TTL（messageDefaultTtlMs）
game.publishMessages(List.of(
    new VKGameMessagePublishCommand(
        VKGameMessageType.SYSTEM_ALERT,
        VKGameMessageScope.PLAYER,
        "player-1",                   // scopeId
        "SYSTEM",                      // senderId
        "异常告警",                      // title
        "检测到异常操作，已记录",           // content
        Map.of("code", "CHEAT_DETECTED"), // payload
        0L                             // expireAtMs（0=默认TTL）
    )
));

消息 Scope 与可见性

VKGameMessageScopescopeId 含义可见对象
GLOBAL（空）所有玩家
LOBBY大厅 ID指定大厅内所有玩家
ROOM房间 ID指定房间内所有玩家
PLAYER玩家 ID仅指定玩家

消息类型

VKGameMessageType系统消息需 ACK说明
PLAYER_CHAT否否玩家聊天消息，senderId 必须为有效玩家 ID
PLAYER_EMOTE否否玩家表情/动作
SYSTEM_NOTICE是否系统公告，senderId 通常为 "SYSTEM"
SYSTEM_ALERT是是需 ACK 确认的告警，未确认时自动重推
SYSTEM_PENALTY是是需 ACK 确认的惩罚通知

匹配系统

提交匹配请求

// 最简请求（仅 playerId + gameType）
String ticketId = game.enqueueMatch(new VKGameMatchRequest("player-1", "chess"));

// 带 rating 和 region（推荐）
String ticketId = game.enqueueMatch(new VKGameMatchRequest(
    "player-1",  // playerId
    "chess",     // gameType
    1200,        // rating（ELO/段位）
    "cn"         // region（区域优先匹配，"" 表示不限区域）
));

// 完整构造（playerId, gameType, rating, region, ratingTolerance, enqueueAtMs）
// ratingTolerance=0 时使用全局 matchmakingBaseRatingTolerance（随等待时间按 matchmakingRatingExpandPerSecond 扩展）
String ticketId = game.enqueueMatch(new VKGameMatchRequest(
    "player-1", "chess", 1200, "cn", 50, System.currentTimeMillis()
));

// 取消匹配，返回 true 表示成功取消（已匹配则无效）
boolean cancelled = game.cancelMatch(ticketId);

// 查询指定 gameType 的当前排队数
int pending = game.pendingMatchCount("chess");

轮询匹配结果（Poll 模式）

// 客户端定时轮询（返回 null 表示结果已过期或 ticketId 无效）
VKGameMatchResult result = game.pollMatchResult(ticketId);
if (result != null && result.getStatus() == VKGameMatchStatus.FOUND) {
    String roomId   = result.getRoomId();
    String joinToken = result.getJoinToken();  // matchJoinRequiresToken=true 时必须用此 token 加入

    // 确认已接受（将状态改为 ACKED，停止重推通知）
    game.ackMatchFound(ticketId, "player-1");

    // 使用 joinToken 加入匹配房间
    game.joinWithToken(roomId, "player-1", joinToken);
}

推送通知模式（无需轮询）

// 绑定匹配结果推送回调（匹配成功后立即回调，未 ACK 时会按退避策略重推）
game.bindMatchNotifier("player-1", result -> {
    if (result.getStatus() == VKGameMatchStatus.FOUND) {
        websocket.send("player-1", Json.toJson(result));
    }
});

// 解绑
game.unbindMatchNotifier("player-1");

手动 Tick（关闭自动调度时）

VostokGame game = Vostok.Game.init(new VKGameConfig().autoStartTicker(false));

// 业务侧手动驱动（tickOnce 为静态方法，直接通过 Vostok.Game 调用）
scheduledExecutor.scheduleAtFixedRate(() -> {
    Vostok.Game.tickOnce();
}, 0, 50, TimeUnit.MILLISECONDS);
Tick 时间门保护
两次 tickOnce() 之间必须间隔至少 1000ms / tickRate（默认 50ms）。若间隔不足，引擎自动跳过本次 Tick，确保逻辑帧率稳定。

持久化与恢复

启用 roomPersistenceEnabled=true 后，引擎每隔 roomSnapshotEveryTicks 帧将房间状态（玩家 Session、attributes、tick 指针）写入磁盘快照。JVM 重启后自动恢复，断线玩家可凭原 sessionToken 重连。

VostokGame game = Vostok.Game.init(new VKGameConfig()
    .roomPersistenceEnabled(true)
    .roomPersistenceDir("./data/vostok-game")
    .roomSnapshotEveryTicks(20)    // 每 20 帧写一次快照
    .roomWalEnabled(true)           // 启用 WAL，减少快照间隙的数据丢失
    .roomRecoveryEnabled(true)      // 启动时自动恢复所有可用快照
);

game.registerLogic("demo", logic);
// 恢复后已有的房间直接可用，勿再次 createRoom
VKGameRoom recovered = game.room("my-room");

枚举说明

VKGameRoomState

枚举值说明
ACTIVE房间正常运行中，可接受玩家加入和指令
DRAINING优雅关闭中，不接受新指令（抛 STATE_ERROR），等待排空后关闭
CLOSED已彻底关闭，从运行时移除

异常说明（VKGameException）

所有操作异常均为 VKGameException（RuntimeException），通过 getCode() 获取错误码。

VKGameErrorCode触发场景
INVALID_ARGUMENT参数非法（null、空字符串、越界）
CONFIG_ERROR配置参数非法
NOT_INITIALIZED模块未初始化时调用操作
ROOM_NOT_FOUND指定 roomId 的房间不存在
ROOM_EXISTS创建房间时 roomId 已存在
ROOM_LIMIT_EXCEEDED全局房间数已达 maxRooms 上限
PLAYER_LIMIT_EXCEEDED房间玩家数已达 maxPlayersPerRoom 上限
LOGIC_NOT_FOUND创建房间时 gameType 对应的 logic 未注册
QUEUE_FULL房间指令队列已满（submit 时）
COMMAND_REJECTED反作弊校验未通过（频率/序号/时间偏移/玩家离线）
MATCHMAKING_ERROR匹配系统未启用或参数非法
STATE_ERROR操作与当前状态不兼容（DRAINING 房间提交指令、Token 已失效等）
