📡 Vostok.Event

Event 事件总线模块

进程内轻量事件总线，支持同步/异步监听器、优先级调度、一次性订阅、Predicate 过滤器，以异步方式发布时监听器互不阻塞。

初始化

Vostok.Event.init();  // 默认配置

Vostok.Event.init(new VKEventConfig()
    .asyncCoreThreads(4)
    .asyncMaxThreads(16)
    .asyncQueueCapacity(10000)
    .listenerErrorStrategy(VKEventListenerErrorStrategy.CONTINUE)  // 某监听失败继续后续
    .rejectionPolicy(VKEventRejectionPolicy.CALLER_RUNS)
);

定义事件类

// 任意 POJO 皆可作为事件
public record OrderCreatedEvent(Long orderId, String userId, BigDecimal amount) {}
public record UserLoginEvent(String userId, String ip, Instant time) {}

订阅事件

同步监听器

VKEventSubscription sub = Vostok.Event.on(OrderCreatedEvent.class, event -> {
    Vostok.Log.info("order {} created", event.orderId());
});

异步监听器

Vostok.Event.on(OrderCreatedEvent.class, VKListenerMode.ASYNC, event -> {
    // 在异步线程池中执行，不阻塞发布者
    sendEmail(event.userId(), "Your order is confirmed");
});

带优先级

// CRITICAL > HIGH > NORMAL > LOW
Vostok.Event.on(UserLoginEvent.class,
    VKListenerMode.SYNC,
    VKEventPriority.HIGH,
    event -> auditLog(event.userId(), event.ip()));

带过滤器

Vostok.Event.on(OrderCreatedEvent.class,
    VKListenerMode.ASYNC,
    VKEventPriority.NORMAL,
    event -> event.amount().compareTo(BigDecimal.valueOf(1000)) > 0,  // 只处理大单
    event -> notifyRiskSystem(event));

一次性订阅

// 触发一次后自动移除
Vostok.Event.once(UserLoginEvent.class, event -> {
    System.out.println("first login: " + event.userId());
});

Vostok.Event.once(UserLoginEvent.class, VKListenerMode.ASYNC, event -> { /* … */ });

发布事件

// 同步发布（等所有同步监听器执行完毕）
Vostok.Event.publish(new OrderCreatedEvent(42L, "user-1", amount));

// 异步发布（立即返回，监听器在后台线程执行）
Vostok.Event.publishAsync(new OrderCreatedEvent(42L, "user-1", amount));

取消订阅

// 取消单个订阅
Vostok.Event.off(sub);

// 移除某事件类型的全部监听器
Vostok.Event.offAll(OrderCreatedEvent.class);

配置参数

参数类型默认值说明
enabledbooleantrue是否启用事件总线
asyncCoreThreadsint4异步线程池核心线程数
asyncMaxThreadsint16异步线程池最大线程数
asyncQueueCapacityint10000异步任务队列容量
asyncKeepAliveMslong60000空闲线程存活时间（ms）
shutdownWaitMslong5000关闭时等待任务完成的最长时间
rejectionPolicyVKEventRejectionPolicyCALLER_RUNS队列满时策略：CALLER_RUNS / DISCARD
listenerErrorStrategyVKEventListenerErrorStrategyCONTINUE监听器异常策略：CONTINUE（继续后续）/ STOP

API 速查

方法说明
on(Class, listener)同步监听（NORMAL 优先级）
on(Class, mode, listener)指定模式监听（SYNC/ASYNC）
on(Class, mode, priority, listener)指定模式 + 优先级
on(Class, mode, priority, filter, listener)带 Predicate 过滤器
once(Class, listener)一次性同步监听
once(Class, mode, listener)一次性异步监听
off(subscription)取消单个订阅
offAll(Class)移除指定事件类型的所有监听器
publish(event)同步发布事件
publishAsync(event)异步发布事件
